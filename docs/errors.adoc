= Errors and API Behavior

This page documents the current Rust renderer entry points and error behavior.
The Rust API is exported from `rust-core/src/lib.rs`; the CLI, C ABI, Python
wrapper, and Java wrapper are thin layers around that library. See
link:packaging.adoc[Packaging and language wrappers] for wrapper packaging and
runtime loading details.

== Entry Points

`Copperlace::from_hocon_str(config)` / `Copperlace::from_hocon_file(path)`::
Parse and compile HOCON input once into a reusable renderer for repeated calls
to `Copperlace::render`.

`Copperlace::render(&self, rule_name: &str) -> Result<String, RenderError>`::
Starts a fresh render for `rule_name` using the previously loaded config.

`RuleSet::from_config(config: hocon_rs::Value) -> Result<RuleSet, RenderError>`::
Compiles a HOCON root value into named render rules and context defaults. The
root must be an object.

`RuleSet::render_rule(&self, rule_name: &str) -> Result<String, RenderError>`::
Starts a fresh render for `rule_name`. Each call gets a new render context and a
new thread-local random number generator.

`render_config_rule(config: hocon_rs::Value, rule_name: &str) -> Result<String, RenderError>`::
Convenience function that compiles the config and renders one rule.

`ruleset_from_hocon_str` / `ruleset_from_hocon_file`::
Parse HOCON input and compile it into a `RuleSet`.

`render_hocon_str` / `render_hocon_file`::
Parse HOCON input and render one rule in a single call.

`RuleSet::from_config_with_processors(config, processors)`::
Compiles a config with the builtin processor registry plus Rust-provided custom
processors. Custom processor names override builtin names when they collide.

== CLI

The package builds a `copperlace` binary:

[source,sh]
----
copperlace render --config <path>
copperlace render -c <path> -r <name> --count <n>
----

The CLI renders `origin` when `--rule` is omitted. It writes rendered output,
help, and version text to stdout. Parse, render, and argument errors are written
to stderr and return a nonzero exit code.

== C ABI

The dynamic library exposes an opaque ruleset handle API:

* `copperlace_ruleset_from_file`
* `copperlace_ruleset_from_string`
* `copperlace_ruleset_render`
* `copperlace_ruleset_free`
* `copperlace_string_free`

Returned strings are allocated by Copperlace and must be released with
`copperlace_string_free`. Ruleset handles must be released with
`copperlace_ruleset_free`.

== Render Context

Each render has an isolated context. Bindings created by `{alias:rule}` and
lazy values created from top-level `context` defaults live only for the current
`render_rule` call.

Bind-if-missing expressions are idempotent within a render. If `{alias:rule}`
runs after `alias` already exists, it leaves the existing value unchanged and
does not render `rule`. Overwrite expressions with `{alias:=rule}` always render
`rule` and replace the value.

The renderer does not currently expose seeded randomness, deterministic replay,
or a way to provide initial context values from the public entry points.

== Error Cases

`UnknownRule(String)`::
Returned when a template reference or requested render rule cannot be resolved
from the current render context, top-level `context` defaults, or named rules.

`UnknownProcessor(String)`::
Returned when a template pipeline references a processor that is not registered.

`ProcessorError { processor, message }`::
Returned when a registered processor rejects a rendered value.

`InvalidExpression(String)`::
Returned when a template expression is malformed, such as `{name | }` or
`{ | uppercase}`.

`EmptyChoice`::
Returned when rendering an array value with no elements.

`CircularRuleReference(Vec<String>)`::
Returned when rule expansion re-enters a rule already on the call stack. The
vector contains the detected path, including the repeated final rule name.

`UnsupportedValue(String)`::
Returned when rendering a value type that compiled to an unsupported node. The
current unsupported value type is `"object"` for nested objects outside the
special top-level `context` object.

`InvalidConfigRoot`::
Returned by `RuleSet::from_config` when the provided HOCON value root is not an
object.

== Current Non-Goals

The current implementation does not define these as supported behavior:

* Escaping literal `{` or `}` characters in templates.
* Weighted choices.
* Seeded or deterministic random output.
* Nested object rendering outside top-level `context`.
