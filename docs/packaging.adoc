= Packaging and Language Wrappers

Copperlace is implemented in Rust and exposed through native and language
wrapper packages.

== Build Workflows

Use the root `Makefile` for normal development workflows:

[source,sh]
----
make check
make package
make clean
----

`make check` runs Rust formatting checks plus Rust, Python, and Java tests.
`make package` builds distributable Python and Java artifacts. `make clean`
removes generated Rust, Python, and Java build output.

== Rust Artifacts

The Rust package in `rust-core/` builds:

* an `rlib` Rust library;
* a `cdylib` native dynamic library;
* the `copperlace` CLI binary.

The CLI supports:

[source,sh]
----
copperlace render --config <path> --rule <name>
----

== C ABI

The native dynamic library exposes an opaque ruleset handle API:

* `copperlace_ruleset_from_file`
* `copperlace_ruleset_from_string`
* `copperlace_ruleset_render`
* `copperlace_ruleset_free`
* `copperlace_string_free`

Returned strings are allocated by Copperlace and must be released with
`copperlace_string_free`. Ruleset handles must be released with
`copperlace_ruleset_free`.

== Python Wrapper

The Python package in `python/` uses `ctypes` over the C ABI. Public entry
points are:

* `RuleSet.from_string(config)`
* `RuleSet.from_file(path)`
* `RuleSet.render(rule)`
* `render_hocon_str(config, rule)`
* `render_hocon_file(path, rule)`

`RuleSet` is a context manager and releases the native handle when closed.
Wrapper failures are raised as `CopperlaceError`.

The wheel build compiles `rust-core` and bundles the platform native library
under `copperlace/native/`.

== Java Wrapper

The Java module in `java/` uses Java FFM over the C ABI. Its Maven coordinates
are `dev.mahe.copperlace:copperlace`. Public entry points are:

* `Copperlace.renderHoconString(config, rule)`
* `Copperlace.renderHoconFile(path, rule)`
* `RuleSet.fromString(config)`
* `RuleSet.fromFile(path)`
* `RuleSet.render(rule)`

`RuleSet` implements `AutoCloseable`; callers should use try-with-resources or
call `close`.

The base Java JAR contains the Java API. Native binaries are packaged in
platform classifier JARs, such as `linux-x86_64`, `linux-aarch64`,
`macos-x86_64`, `macos-aarch64`, `windows-x86_64`, and `windows-aarch64`.

At runtime, the Java wrapper loads the native library in this order:

. `COPPERLACE_LIBRARY_PATH`
. packaged classifier resource
. local Rust build output for source-tree development
